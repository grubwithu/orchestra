# 约束类型细分方案

## 原始约束类型 (5个)

1. `CT_VALUE_COMPARISON` - 值比较
2. `CT_BITWISE_OPERATION` - 位运算
3. `CT_STRING_MATCH` - 字符串匹配
4. `CT_ARITHMETIC_OPERATION` - 算术运算
5. `CT_COMPOUND_OPERATION` - 复合运算

## 细分后的约束类型 (10个)

### 1. 值比较类
- **`CT_EQUALITY_COMPARISON`** (`eq_cmp`)
  - **描述**: 等值比较
  - **操作符**: `==`, `!=`
  - **示例**: `a == b`, `x != 0`

- **`CT_RELATIONAL_COMPARISON`** (`rel_cmp`)
  - **描述**: 关系比较
  - **操作符**: `<`, `>`, `<=`, `>=`
  - **示例**: `a > b`, `x <= 100`

### 2. 运算类
- **`CT_ARITHMETIC_BASIC`** (`arith_basic`)
  - **描述**: 基础算术运算
  - **操作符**: `+`, `-`, `*`, `/`, `%`
  - **示例**: `a + b > 0`, `x % 2 == 0`

- **`CT_BITWISE_OPERATION`** (`bit_opr`) (保留)
  - **描述**: 位运算
  - **操作符**: `&`, `|`, `^`, `~`, `<<`, `>>`
  - **示例**: `a & b != 0`, `x << 2 > 0`

- **`CT_LOGICAL_OPERATION`** (`logical_opr`) (新增)
  - **描述**: 逻辑运算
  - **操作符**: `&&`, `||`, `!`
  - **示例**: `a && b`, `!flag`, `x || y`

### 3. 字符串类
- **`CT_STRING_EXACT_MATCH`** (`str_exact`)
  - **描述**: 精确字符串匹配
  - **函数/操作**: `strcmp()`, `strncmp()`, `memcmp()`, `==`
  - **示例**: `strcmp(a, b) == 0`, `s == "hello"`

- **`CT_STRING_PATTERN_MATCH`** (`str_pattern`)
  - **描述**: 模式字符串匹配
  - **函数**: `strstr()`, 子串查找
  - **示例**: `strstr(s, "sub") != NULL`

### 4. 复杂类型
- **`CT_POINTER_COMPARISON`** (`ptr_cmp`) (新增)
  - **描述**: 指针比较
  - **操作**: 指针解引用比较, `NULL`检查, 指针算术
  - **示例**: `*ptr == value`, `ptr != NULL`, `ptr1 > ptr2`

- **`CT_TYPE_CONVERSION`** (`type_conv`) (新增)
  - **描述**: 类型转换和检查
  - **操作**: `sizeof()`, 类型转换, `typeid`, C++ cast操作
  - **示例**: `(int)value == 0`, `sizeof(type) > 0`

- **`CT_COMPOUND_CONDITION`** (`comp_cond`) (原复合运算更名)
  - **描述**: 复合条件
  - **特征**: 混合多种操作, 复杂表达式
  - **示例**: `(a + b) > (c * d)`, `a == b && c > d`

## 检测逻辑说明

### 优先级顺序 (从上到下)
1. **字符串操作** - 最高优先级
2. **指针操作** - 避免与乘法运算混淆
3. **类型转换** - 检查类型相关操作
4. **逻辑运算** - 检查逻辑连接符
5. **比较运算** - 区分等值和关系比较
6. **算术运算** - 基础数学运算
7. **位运算** - 位级操作
8. **复合条件** - 默认/兜底分类

### 关键改进点

1. **更细粒度的分类**: 从5类扩展到10类，提供更精确的分析
2. **消除歧义**: 明确区分等值比较和关系比较
3. **新增专业类别**: 增加指针、类型转换等专业领域的约束类型
4. **保留兼容性**: 保持原有核心分类，只进行扩展和细化
5. **逻辑优化**: 改进检测算法，避免错误分类
